# Правила сообщений для QA task tracker

## Создание задачи

Бот создает новую задачу только из отдельного сообщения с идентификатором задачи и рабочими ссылками или инструкцией.

Поддерживаемый формат:

```text
@diiaanag @glebc0re @bbbbbbtp
idApp 320

репа:
https://github.com/AezakmiFastApps/chat_320

ТЗ:
https://docs.google.com/document/d/...

ФИГМА:
https://www.figma.com/design/.../idFastApp-320.--AI-Chat-with-ILama-World--generator-

2h
```

Что важно:

- в сообщении должен быть id задачи: `idApp 320`, `image_212`, `video_218`, `chat_320`;
- Figma/GitHub/ТЗ помогают боту заполнить карточку;
- исполнитель не назначается по тегу, исполнитель назначается реакцией тестировщика на исходное сообщение задачи;
- если задача с таким ключом уже открыта, сообщение добавится как обновление к существующей задаче;
- если старая задача с таким ключом уже завершена, новое сообщение создаст новую задачу.

## История задачи

Бот учитывает отчеты, фиксы, вопросы и закрытие только внутри reply-цепочки задачи.

Разрешенные варианты:

- ответ на исходное сообщение задачи;
- ответ на сообщение, которое уже попало в историю этой задачи.

Сообщения вне reply-цепочки не меняют статус задачи и не добавляются в историю, даже если похожи на фикс или закрытие.

## Отчет

Отчет отправляется ответом в историю задачи и считается отчетом только если в сообщении есть архив `.zip`, `.rar` или `.7z`.

Примеры:

```text
@developer привет, вот отчет
report_320.zip
```

```text
@developer
idapp_320_report.zip
```

Если архив прикреплен файлом Telegram, бот увидит имя файла и добавит событие как отчет.

## Замечания

Замечания отправляются ответом в историю задачи.

Примеры:

```text
@developer указал, что не исправлено + добавил новые пункты
```

```text
@developer указал, что не исправлено
```

Google Docs больше не считается отчетом. Другие ссылки тоже не считаются отчетом, если нет архива.

## Фикс или обновление

Фикс отправляется ответом в историю задачи.

Примеры:

```text
@tester фиксы
```

```text
@tester перезалил
```

Также как обновление считаются вложения с архивами `.zip`, `.rar`, `.7z`, фото и видео, если они отправлены ответом в историю задачи.

## Закрытие задачи

Закрытие работает только коротким ответом внутри истории задачи.

Разрешенные фразы:

```text
все окей
всё окей
готово
сделано
закрыто
```

Длинные сообщения со словами вроде "закрыли одну итерацию" не закрывают задачу.

## Ручная смена статуса в чате

Нужно тегнуть бота. Самый удобный формат - ответить на задачу или любое сообщение из ее истории.

Примеры:

```text
@task_tracker_qa_bot поставь статус в работе
@task_tracker_qa_bot поставь статус завершено
@task_tracker_qa_bot поставь статус пауза
```

Если нет reply, можно указать id задачи в сообщении:

```text
@task_tracker_qa_bot статус idapp_320 в работе
@task_tracker_qa_bot статус idapp_320 завершено
@task_tracker_qa_bot статус idapp_320 пауза
```

## Статус по задачам за период

Нужно тегнуть бота и указать период.

Примеры:

```text
@task_tracker_qa_bot дай статус задач с 01.06 по 10.06
@task_tracker_qa_bot дай статус по задачам за 2026-06-01 2026-06-10
@task_tracker_qa_bot дай статус задач с 6 июня по 30 июня
```

Если год не указан, бот использует текущий год по Москве.
Сводка показывает только задачи того чата, где ее запросили.

## Реакции бота

Бот почти не пишет сообщения об успешных действиях, чтобы не засорять чат. Вместо этого он ставит реакции:

- `👍` - новая задача создана;
- `👀` - сообщение добавлено в историю задачи;
- `🙏` - принят отчет-архив;
- `✍️` - добавлены замечания;
- `⚡` - добавлен фикс или обновление;
- `🏆` - задача закрыта;
- `💯` - команда боту выполнена.

Если Telegram не принимает основную реакцию в чате, бот пробует запасную реакцию.
